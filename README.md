# Lyft Trip Data

## Overview

This project explores a sample Lyft database using SQL to answer common business questions about riders, trips, and vehicles.

The project demonstrates how relational databases can be queried using joins, unions, common table expressions (CTEs), aggregate functions, and sorting techniques.

---

## Database Tables

The database contains three primary tables:

- **trips**
- **riders**
- **cars**

The project explores the relationships between these tables to analyze ride activity.

---

- Aggregate functions
  - AVG()
  - COUNT()
- ORDER BY
- LIMIT

---

## Queries Included

- Explore database tables
- Identify primary keys
- Create a Cartesian product using CROSS JOIN
- Combine rider and trip information
- Join trip and vehicle data
- Merge rider tables using UNION
- Calculate average trip cost
- Find riders with fewer than 500 trips
- Count active vehicles
- Identify the two most-used vehicles
